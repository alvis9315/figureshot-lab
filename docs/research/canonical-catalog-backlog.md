# 公共模型目錄構想(Backlog,討論中)

> 2026-07-12 由擁有者提出,源自實際使用經驗(海外觀眾常反覆詢問「這是哪一隻/哪個品牌」)。
> 狀態:**討論中**——不阻擋現行 PRE 閘門與 Wave 進度,不變更已定案的 ADR/data-model(MVP 仍用 spec 11 節簡化 schema)。

## 問題

- 現行 MVP 決策(spec 19 節)是使用者**自由輸入文字**建立收藏,容易打錯名稱、缺欄位,甚至誤植盜版商品資訊。
- 擁有者本人的實際痛點:海外觀眾常反覆詢問「這是哪一隻模型/哪個品牌」——顯示模型辨識資訊在社群互動中被高頻需要,自由文字輸入撐不起「自動顯示辨識資訊」這件事。
- 盜版/侵權商品若被當正常收藏上傳並公開展示,可能傷害社群風氣,也與 spec 20G(IP 與內容治理)的既定立場衝突。

## 擁有者提案

新增收藏時**限定從既有分類 + 已發行模型清單中挑選**(而非自由輸入),達成:

1. 降低新手建錯資料的風險(辨識資訊天生正確)。
2. 天然過濾盜版(清單只收錄官方已發行商品)。
3. 未來可在分享頁/生成結果自動顯示「品牌/系列」,減少被重複詢問。

## 風險評估(提前想清楚,不是要澆冷水)

| 風險 | 說明 | 影響 |
|---|---|---|
| 維護是持續成本 | 新品發售頻率高(Bandai/GSC/壽屋/Hot Toys/盒玩品牌…),目錄要跟得上才有用 | 目錄跟不上 = 新手買的新品反而找不到,體驗反效果 |
| 長尾覆蓋不完 | 客製改造、絕版老物、小廠/同人商品、非日系品牌 | 若**強制**選單挑選,會擋住這些使用者建立收藏 |
| 影響 Activation | 收藏建立是啟用漏斗第一步 | 目錄不完整 + 強制限定 = 直接衝擊剛下修到 35% 的 Activation 門檻(PRE-010) |
| 爬蟲抓取有法遵界線 | 本 repo 爬蟲類專案既有紀律:不繞驗證、不高頻請求、僅個人分析用途 | 大量商業性抓取廠商資料轉載到自有資料庫,需先確認各官網 ToS |

## 建議方向(混合制,尚未定案)

不做「全有全無」的二選一:

1. 新增收藏時**優先讓使用者從目錄搜尋/自動完成**(打字「五條悟」跳出已知商品供選)。
2. **目錄找不到時允許「自訂新增」**,但標記為「待系統比對」狀態(之後人工或演算法合併進主目錄,類似維基百科的詞條合併機制)。
3. 盜版防範**不能只靠目錄機制自動過濾**——目錄外的自訂新增仍可能被亂填,建議額外在使用條款/社群規範明訂禁止上傳盜版商品,違規可檢舉下架。這是治理規則,不是純技術問題。

## 待研究:資料來源與更新方式(列選項,不做決定)

| 選項 | 說明 | 待確認 |
|---|---|---|
| 官方廠商網站/型錄 | Bandai、Good Smile、壽屋等官網新品頁 | 各家 ToS 是否允許抓取;更新頻率 |
| 電商平台商品頁 | 駿河屋、AmiAmi、HobbyLink Japan | 同上;可能有商業轉載限制 |
| 既有資料庫(如 MFC) | 是否有開放 API/資料匯出 | 需查是否提供、是否允許此用途 |
| 使用者提交 + 審核 | 玩家自己貢獻缺漏商品 | 需要審核機制與人力,呼應 spec 20C 的公開資料治理章節 |
| 廠商官方 RSS/新聞稿 | 較穩定但覆蓋率有限(通常只有大廠) | 適合作為「重點新品」自動化來源之一,非全量方案 |
