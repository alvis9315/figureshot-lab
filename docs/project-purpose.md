# Project Purpose

> 摘要版;完整脈絡見 product-spec.md(凍結正本)。

## 背景與問題

模型玩家與收藏家的痛點不是「不會用 AI 寫文案」,而是:不知道今天拍哪隻、不知道模型怎麼搭、不知道拍什麼主題、不知道怎麼把作品變成社群內容。收藏往往「收了就只放著」。

## 目標

把「選模型 → 抽主題 → 生成優化 → 擺拍建議 → 文案輸出」做成完整工作流,讓收藏被重新使用、創作門檻下降、社群分享變容易。

## 不做什麼(MVP)

- 全世界模型公開資料庫(使用者自建收藏起步)
- 付款/訂閱、B 端報表、推薦機器學習、AI 圖片生成
- 台灣限定平台綁定(Global-first,en + zh-TW)

## MVP 範圍

使用者自建收藏庫 + 靈感輪盤 + AI 擺拍優化 + IG 文案(P0),登入/文案/紀錄(P1)。

## 使用者角色

模型玩家、收藏家、模型攝影師、社群經營者(C 端);品牌/店家為長期 B 端方向(Stage 4)。

## 核心流程

新增收藏 → 貼標籤 → 輪盤抽 1~3 隻 + 場景/情緒/任務 → 生成主題與擺拍建議 → 儲存/分享。

## 未來擴充

社群期(每週挑戰、投稿)→ 資料期(趨勢統計)→ 商業合作期(品牌挑戰、導購、趨勢報告)。詳見 roadmap.md 與 spec 17 節。
